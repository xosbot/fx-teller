01Executive Summary

A bird's-eye view of what we are building, why, and how.

What is FX-Teller?

FX-Teller is a live podcast-style trading community built for forex traders in Kerala, India. A single host streams live audio commentary on market sessions and pushes real-time trade calls to listeners — with instant alerts, sentiment-driven music cues, and a simple subscription model.

Phase 1 Strategy: Web-First

Rather than building a native mobile app (which requires App Store approval, platform-specific development, and weeks of testing), Phase 1 launches as a browser-based web application. This approach:

  • Gets the product to market in 4 weeks vs. 8–12 weeks for mobile
  • Works instantly on any device — phone, tablet, laptop — no install required
  • Lets us validate with real users before investing further
  • Reuses the existing Host Console and NestJS backend already built

Why This Approach Wins

Every day we are not live is a day we are not learning. A web-first Phase 1 means we can sign up real traders, collect feedback, and prove the model — all before committing to native mobile app development in Phase 2. The mobile app will share the same backend, so no work is wasted.

Core value proposition

Live Audio

Real-time commentary via 100ms.live. Listen through your browser — no downloads, no apps.

Instant Calls

Trade calls arrive in under a second with a browser notification. Pair, direction, entry, SL, TP — all on screen.

Sentiment Cues

Background music switches between Calm and Alert. You feel the market before you read it.

02Project Scope

Exactly what Phase 1 includes — and what it does not.

1. Listener Web App Build

A browser-based interface where traders join live sessions, hear audio commentary, and receive trade calls.

  • Phone OTP authentication — no password needed
  • Live audio room via 100ms embedded player
  • Real-time trade call cards with pair, direction, entry, SL, TP
  • Browser push notifications for incoming trade calls
  • Session schedule with upcoming trading sessions
  • Trial countdown banner for the 3-day free trial
  • Razorpay checkout for subscription (pricing to be decided)
  • Music sentiment indicator — Calm / Alert state
  • User profile with name, phone, and subscription status
  • Responsive, mobile-first design that works on all screen sizes
  • PWA basics for "Add to Home Screen" on supported browsers
  • Auto-reconnect on network drop

2. Host Console Built

The host's web dashboard for running live sessions. Already built — minor polish and deployment.

  • Start and stop live sessions
  • Push trade calls with instrument, side, entry, SL, TP
  • Switch music sentiment between Calm and Alert
  • View recent trade call history

3. Backend API Complete & Deploy

NestJS backend powering all application modules. Already scaffolded; needs completion and deployment.

Auth & Users

  • Phone OTP request and verification
  • JWT session management
  • User profile and role management

Sessions & 100ms

  • Create and manage live sessions
  • 100ms token minting for host and listener roles
  • SSE event broadcast bus for real-time updates

Trade Calls & Music

  • Push trade calls with real-time broadcast
  • Music sentiment state switching
  • Recent calls history API

Subscriptions

  • Razorpay subscription creation
  • Webhook verification handler
  • Trial state management

Deferred to Phase 2+ (Out of Scope)

  • Native mobile apps (iOS / Android)
  • In-app chat between listeners and host
  • Multi-host marketplace
  • Advanced analytics dashboard
  • Referral program
  • Yearly subscription plans
