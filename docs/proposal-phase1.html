01Executive Summary

A bird's-eye view of what we are building and why.

What is FX-Teller?

FX-Teller is a live podcast-style trading community built for forex traders in Kerala, India. A single host streams live audio commentary on market sessions and pushes real-time trade calls to listeners — with vibration alerts, sentiment-driven music cues, and a simple subscription model.

Phase 1 Strategy: Web-First

Instead of building a native mobile app that requires App Store approval, device-specific development, and lengthy testing cycles, we are launching Phase 1 as a browser-based web application. This approach:

  • Gets the product to market in 4 weeks vs. 8–12 weeks for a mobile app
  • Works on any device (phone, tablet, laptop) — no install needed
  • Allows us to validate with initial clients before investing in mobile
  • Uses the already-built Host Console web app and NestJS backend as a foundation

Why This Approach Wins

Every day we are not live is a day we are not learning. A web-first Phase 1 lets us sign up real users, collect feedback, and prove the model — all before committing to the complexity of native app development in Phase 2. The mobile app will use the same backend, so nothing is wasted.

Core value proposition

🎙 Live Audio

Real-time commentary via 100ms.live. Listen through your browser — no app, no downloads.

📳 Instant Calls

Trade calls arrive in <1s with a browser notification. Pair, direction, entry, SL, TP — all on your screen.

🎵 Sentiment Cues

Background music switches between Calm and Alert. You feel the market before you read it.

02Project Scope

Exactly what Phase 1 includes — and what it does not.

1. Listener Web App Build

A browser-based interface where traders join live sessions, hear audio, and receive trade calls.

  • Phone OTP authentication (no password)
  • Live audio room via 100ms embedded player
  • Real-time trade call cards with pair, direction, entry, SL, TP
  • Browser push notifications for trade calls
  • Session schedule with upcoming sessions
  • Trial countdown banner (3-day free trial)
  • Razorpay checkout for subscription (₹499/month)
  • Music sentiment indicator (Calm / Alert)
  • Basic user profile (name, phone, subscription state)
  • Responsive design (mobile-first, works on all screens)
  • Progressive Web App (PWA) basics for "add to home screen"
  • Auto-reconnect on network drop

2. Host Console Built

The host's web dashboard — already built in the monorepo. Minor UI polish and deployment.

  • Start / stop live sessions
  • Push trade calls (instrument, side, entry, SL, TP)
  • Switch music sentiment (Calm / Alert)
  • Recent trade call history table

3. Backend API Complete & Deploy

NestJS backend powering all modules. Already scaffolded; needs completion, testing, and deployment.

Auth & Users

  • Phone OTP request + verify
  • JWT session management
  • User profile & role management

Sessions & 100ms

  • Create / manage live sessions
  • 100ms token minting (host + listener)
  • SSE event broadcast bus

Trade Calls & Music

  • Push trade calls with broadcast
  • Music sentiment state switching
  • Recent calls history endpoint

Subscriptions

  • Razorpay subscription creation
  • Webhook verification handler
  • Trial state management

⛔ Out of Scope (Phase 2+)

  • Native mobile app (iOS / Android)
  • In-app chat
  • Multi-host marketplace
  • Advanced analytics dashboard
  • Referral program
  • Yearly subscription plans
